# 파일 검색 도구 가이드 — find, grep, rg

현재 경로 및 모든 하위 경로에서 파일을 찾는 방법을 정리한다.

---

## 1. 파일 **이름**으로 찾기 — find

```bash
# 기본: 현재 디렉토리 + 모든 하위 디렉토리
find . -name "*키워드*"

# 확장자 지정
find . -name "*config*.yaml"

# 대소문자 무시
find . -iname "*readme*"
```

### find vs glob 패턴 차이

| 방법            | 비유                                  |
| --------------- | ------------------------------------- |
| `find -name`    | 도서관에서 **책 제목**에 키워드가 있는 책 찾기 |
| `grep -r`       | 도서관에서 **책 내용**에 키워드가 언급된 책 찾기 |

---

## 2. 파일 **내용**으로 찾기 — grep / rg

### grep (기본 내장)

```bash
# 재귀 검색 (-r 필수)
grep -r "키워드" .

# 파일 경로만
grep -rl "키워드" .

# 특정 확장자만
grep -r "키워드" . --include="*.py"
```

### rg (ripgrep) — 더 빠른 대안

```bash
# 기본이 재귀 검색 (플래그 불필요)
rg "키워드"

# 특정 파일 타입
rg "키워드" --type py
```

---

## 3. rg (ripgrep) 상세

### rg란?

**ripgrep**의 CLI 명령어. Rust로 작성된 초고속 텍스트 검색 도구.

### grep vs rg 비교

| 항목               | `grep -r`          | `rg` (ripgrep)         |
| ------------------ | ------------------ | ---------------------- |
| 속도               | 느림               | 훨씬 빠름 (멀티스레드) |
| `.gitignore` 존중  | ❌ 수동 설정       | ✅ 자동으로 무시        |
| `node_modules` 등  | 다 뒤짐            | 자동 스킵              |
| 출력 가독성        | 밋밋함             | 컬러 + 줄번호          |
| 설치               | 기본 내장          | 별도 설치 필요         |

### 설치

```bash
brew install ripgrep
```

### 기본 동작: 재귀 검색

```bash
# grep은 -r 필요, rg는 기본이 재귀
grep "키워드"       # 현재 디렉토리만
grep -r "키워드"    # 하위 포함
rg "키워드"         # 하위 포함 (기본)
```

### 자동 무시 규칙

```
✅ 검색함              ❌ 자동 무시
─────────────         ─────────────
소스코드 파일          .gitignore에 명시된 파일
설정 파일              .git/ 디렉토리
텍스트 파일            바이너리 파일 (이미지, 실행파일 등)
                      node_modules/ (gitignore에 있으면)
```

무시된 파일까지 뒤지려면:

```bash
rg --no-ignore "키워드"       # .gitignore 무시하고 검색
rg --no-ignore -. "키워드"    # 숨김파일(.dotfiles)까지 포함
```

## 4. rg 다중 키워드 검색

### OR 검색 (하나라도 포함)

```bash
# 정규식 OR
rg "키워드1|키워드2|키워드3"

# -e 플래그 (가독성 좋음)
rg -e "키워드1" -e "키워드2" -e "키워드3"

# 예시
rg "TODO|FIXME|HACK"
```

### AND 검색 (모두 포함된 파일)

```bash
# 같은 줄에 모두 있을 때
rg "키워드1" | rg "키워드2"

# 파일 안 어디든 모두 포함 (-l: 파일 단위)
rg -l "키워드1" | xargs rg -l "키워드2"

# 3개 이상
rg -l "키워드1" | xargs rg -l "키워드2" | xargs rg -l "키워드3"
```

### 비유

| 방식                | 비유                                      |
| ------------------- | ----------------------------------------- |
| OR (`\|`)           | "피자 **또는** 치킨 파는 가게 찾아줘"     |
| AND (파이프 체이닝) | "피자 **그리고** 치킨 **둘 다** 파는 가게 찾아줘" |

---

## 5. 유용한 옵션 요약

```bash
# 파일 경로만 출력
rg -l "패턴"

# 매칭 개수만
rg -c "패턴"

# 컨텍스트 (전후 N줄)
rg -C 3 "패턴"

# 특정 디렉토리에서
rg "패턴" ./src/

# 정규식 활용
rg "func\s+\w+" --type go
```
