# 네트워크 대역폭 단위 변환 완전 정복: Gbit, GB, GiB

## 출발점: 25GbE로 초당 몇 바이트를 보낼 수 있나?

파일 크기는 Byte(바이트)로 표시하니까, bit를 Byte로 바꿔야 한다.

```
공식: Byte = bit ÷ 8    (1 Byte = 8 bits)

계산:
  25 Gbit/s ÷ 8 = 3.125 GB/s
```

> 현대 표준에서 1 byte = 8 bits이나, 역사적으로 byte 크기가 달랐던 시기도 있다 (CDC 6600의 6-bit 등). 현재는 IEEE 1541-2002에서 1 byte = 8 bits로 표준화되었다.

주요 이더넷 속도 환산표:

| 규격     | 계산     | 이론적 최대값             | 실효 처리량 (~95-97%)   |
| -------- | -------- | ------------------------- | ----------------------- |
| 1 GbE    | 1 ÷ 8   | 0.125 GB/s = 125 MB/s    | ~119-121 MB/s           |
| 10 GbE   | 10 ÷ 8  | 1.25 GB/s                | ~1.19-1.21 GB/s         |
| 25 GbE   | 25 ÷ 8  | 3.125 GB/s               | ~2.97-3.03 GB/s         |
| 100 GbE  | 100 ÷ 8 | 12.5 GB/s                | ~11.9-12.1 GB/s         |

> **검증 결과 (CONFIRMED, HIGH)**: 위 수치는 이론적 최대값이다. 실제 네트워크 전송에서는 이더넷 프레임 헤더(프리앰블, IFG, CRC 등 ~2.5%), TCP/IP 헤더 등 프로토콜 오버헤드가 존재하여 실효 처리량은 이론값의 약 95~97%이다. (IEEE 802.3by)

이 계산에서 `25 ÷ 8`이 깔끔하게 되는 이유는 **양쪽 다 같은 SI(10진법) 기준**이기 때문이다:

```
25 × [10^9 bit/s] ÷ 8 ÷ [10^9]
     ↑ Gbit의 G        ↑ GB의 G
     둘 다 10^9         10^9끼리 약분!

= 25 ÷ 8 = 3.125   ← G가 사라지고 숫자만 남음
```

> 비유: "25km를 몇 km냐"고 물으면 그냥 25인 것처럼 — 같은 접두어끼리는 약분되니까 깔끔하다.

## 두 가지 단위 체계: SI vs 2진법

### SI 접두어 (10의 거듭제곱) — 네트워크, 하드디스크 제조사

네트워크의 **모든** 접두어(K, M, G, T)는 일관되게 SI 기준이다.

| 접두어   | 의미 | 값                        |
| -------- | ---- | ------------------------- |
| K (Kilo) | 10^3 | 1,000                     |
| M (Mega) | 10^6 | 1,000,000 (백만)          |
| G (Giga) | 10^9 | 1,000,000,000 (10억)      |

> **검증 결과 (CONFIRMED)**: 1 Gbit = 10^9 bit. BIPM SI 접두어 정의 및 IEEE 802.3에서 확인.

### 2진 접두어 (2의 거듭제곱) — OS, 메모리

| 접두어   | 의미 | 값              |
| -------- | ---- | --------------- |
| Ki (Kibi) | 2^10 | 1,024           |
| Mi (Mebi) | 2^20 | 1,048,576       |
| Gi (Gibi) | 2^30 | 1,073,741,824   |

> **검증 결과 (CONFIRMED)**: GiB = 2^30 byte. IEC 60027-2, IEC 80000-13 국제 표준에서 확인.

### 접두어별 SI와 2진법의 차이율

| 접두어   | SI     | 2진법  | 차이율  |
| -------- | ------ | ------ | ------- |
| K / Ki   | 1,000  | 1,024  | ~2.4%   |
| M / Mi   | 10^6   | 2^20   | ~4.9%   |
| G / Gi   | 10^9   | 2^30   | ~7.4%   |
| T / Ti   | 10^12  | 2^40   | ~10.0%  |

> 접두어가 커질수록 차이가 벌어진다. "항상 약 7%"가 아니다.

## GB vs GiB: 같은 데이터, 다른 표기

GB가 **두 가지 의미**로 쓰이는 혼란을 해결하려고 GiB가 만들어졌다:

```
하드디스크 제조사: "1 GB = 1,000,000,000 byte" (SI, 10의 거듭제곱)
운영체제(OS):     "1 GB = 1,073,741,824 byte" (2의 거듭제곱)

→ 같은 "GB"인데 양이 다르니까 혼란!
→ 해결: 2진법 기준은 "GiB"라고 부르자!
```

| 표기            | 풀네임        | 기준                                   |
| --------------- | ------------- | -------------------------------------- |
| **Gbit** (= Gb) | Giga**bit**   | SI (10^9 = 10억 **bit**)              |
| **GB**          | Giga**byte**  | SI (10^9 = 10억 **byte**)             |
| **GiB**         | Gibi**byte**  | 2진법 (2^30 = 1,073,741,824 **byte**) |

Gbit과 GiB는 둘 다 정보량이라는 **같은 차원**이지만, bit→byte 변환과 SI→2진 접두어 변환이라는 **두 단계**를 거쳐야 한다.

> **검증으로 수정된 비유**: 원래 "kg과 마일처럼 완전히 별개"라고 설명했으나, kg(질량)과 마일(거리)은 차원 자체가 다르다. Gbit과 GiB는 동일 차원(정보량)이므로 이 비유는 부정확하다. 더 정확한 비유: 마일과 킬로미터처럼 같은 거리를 다른 체계로 표현하되, 추가로 단위(bit vs byte)도 다른 상황.

### 실제 변환 예시

```
3,125,000,000 byte (실제 byte 수, 하나의 사실)

표기법 A: 3,125,000,000 ÷ 1,000,000,000 = 3.125 GB   (SI)
표기법 B: 3,125,000,000 ÷ 1,073,741,824 = 2.91 GiB   (2진법)

같은 데이터, 다른 표기
```

> 비유: 물 1리터를 "1,000ml"이라고 부르든 "약 33.8oz"라고 부르든 물의 양은 같은 것처럼.

### 1TB HDD가 OS에서 "931GiB"로 보이는 이유

```
제조사: 1 TB = 1,000,000,000,000 byte (SI)
OS:     같은 byte 수 ÷ 1,073,741,824 = 931.32 GiB

→ OS가 "931 GB"라고 표시하지만, 실제로는 931 GiB
→ 대부분의 OS가 GiB 값을 "GB"라고 잘못 표시하는 것이 혼란의 원인
```

> **검증 결과 (CONFIRMED)**: 산술 정확 (931.32...). 주의: macOS는 10.6(Snow Leopard)부터 SI 표기로 전환했고, Windows는 여전히 GiB를 GB로 표시한다.

## 왜 네트워크와 하드디스크는 10진법을 쓰나?

### 네트워크: 통신 업계의 역사 + 물리적 합리성

```
전화선 시대 (1800년대~)
→ 전신, 전화, 모뎀 — 모두 통신 엔지니어가 만듦
→ 통신 엔지니어는 물리학/전기공학 출신
→ 물리학은 당연히 SI 단위 (10의 거듭제곱) 사용
→ 그래서 baud rate, bps 전부 10진법 기준
```

나중에 컴퓨터가 네트워크를 쓰게 됐지만, 네트워크 전송은 RAM과 달리 2의 거듭제곱에 물리적 의존성이 없으므로 SI 단위가 기술적으로도 적절하다. 물리 계층의 클럭 주파수(Hz)가 SI 단위인 것도 자연스러운 연장이다.

> **검증 결과 (LIKELY, MEDIUM)**: "통신 업계 유산"이라는 설명의 방향은 맞지만, 단일 원인 귀속은 단순화이다. 실제로는 (1) 통신 역사적 관행, (2) 물리 계층 클럭 주파수가 SI 기반, (3) 네트워크 전송이 2의 거듭제곱에 비의존적 — 이라는 복합 이유가 작용한다.

### 하드디스크: SI 표준 준수 + 물리구조 부합 + 마케팅 이점

> **검증으로 수정된 내용 (LIKELY, HIGH)**: 원래 "마케팅 때문"이라고 단정했으나, 이는 과도한 단순화이다.

HDD 제조사가 SI 10진 단위를 쓰는 이유는 복합적이다:

1. **역사적 선행**: HDD 제조사는 **1970년대부터** SI 10진 접두어를 사용해왔다. IEC가 이진 접두어(KiB, MiB, GiB)를 정의한 1998년보다 **20년 이상 앞선다**.
2. **물리적 구조 부합**: HDD는 RAM과 달리 연속적 자기 기록 방식으로 데이터를 저장한다. 섹터 수, 트랙 수, 플래터 수가 2의 거듭제곱일 필요가 없어 SI 단위가 자연스럽다.
3. **국제 표준 적합**: IEC 80000-13, IEEE 1541-2002 모두 SI 접두어의 10진 사용을 표준으로 인정한다.
4. **마케팅 이점**: 결과적으로 10진법 표기가 숫자를 더 크게 보이게 하는 것은 사실이다.

```
같은 디스크를 표기할 때:
2진법: 931 GiB
10진법: 1,000 GB  ← 숫자가 더 크다
```

실제로 WD, Seagate 대상 소비자 소송이 있었으나, 모두 **합의(settlement)로 종결**되었고 법원이 "기만"으로 판정한 적은 없다. 제조사는 "SI 표준을 따랐다"고 방어했으며, 이는 법적으로 유효한 주장이다.

> 정리: SI 표준 준수가 기술적으로도, 법적으로도 타당한 선택이었고, 마케팅 이점은 부수적 효과이다. "의도적 기만"이라고 단정할 근거는 부족하다.

### RAM은 왜 2진법인가?

> **검증으로 수정된 내용 (LIKELY, HIGH)**: 원래 "물리적으로 만들 수 없다"고 했으나, 이는 과장이다.

RAM 칩의 용량이 2의 거듭제곱인 이유는 **구조적 강제** 때문이다:

```
메모리 셀은 주소 라인(address line)으로 접근
→ n비트 주소선 = 정확히 2^n개의 셀에 접근
→ 주소 라인 1개 추가 = 용량 2배
→ 10개 = 2^10 = 1,024
→ 20개 = 2^20 = 1,048,576
```

2의 거듭제곱이 아닌 용량을 만들면 주소 공간의 일부가 낭비된다. 물리 법칙이 금지하는 것은 아니지만, 세 가지 힘이 결합하여 **사실상 만들어지지 않는다**:

1. **이진 주소 디코더 구조**: 비이진 용량은 실리콘 면적 낭비
2. **JEDEC 표준**: DDR3/4/5 모두 2의 거듭제곱 용량만 명시
3. **경제성**: 비표준 용량은 호환성과 경제성을 모두 상실

> 참고: 12GB/48GB 같은 시스템 총 용량은 **여러 개의 2의 거듭제곱 칩을 조합한 결과**이다. Samsung 3GB LPDDR3(2014)도 내부는 6개의 4Gb 다이를 적층한 것이다. 개별 칩 자체가 비2의거듭제곱인 상용 사례는 없다.

반면 하드디스크와 네트워크는:

- 하드디스크: 플래터 위에 비트를 **연속적으로** 기록 → 용량이 꼭 2의 배수일 필요 없음
- 네트워크: 전선 위에 신호를 **연속적으로** 보냄 → 속도가 꼭 2의 배수일 필요 없음

## 실무에서는 어떻게 쓰나?

**네트워크 속도를 말할 때는 보통 SI(GB)로 통일**한다.

```
✅ 일반적인 표현:
  "25GbE면 이론상 최대 3.125 GB/s"

🤔 굳이 이렇게 안 함:
  "25GbE면 이론상 최대 2.91 GiB/s"
```

GiB가 의미 있는 순간은 **OS에서 실제 전송 속도를 모니터링할 때**:

```
$ iperf3 결과: 2.85 GiB/s   ← OS가 2진법으로 보여줌
네트워크 스펙: 3.125 GB/s    ← 제조사가 10진법으로 표기

→ "어? 스펙보다 느린데?"
→ 아닌데, 단위가 다른 것뿐!
```

| 상황                       | GB (SI)        | GiB (2진법)          |
| -------------------------- | -------------- | -------------------- |
| 네트워크 스펙 비교         | **이걸 쓰세요** | 불필요               |
| OS 모니터링 화면과 비교    | 변환 필요       | **화면에 이걸로 나옴** |
| 실제 데이터 양             | 둘 다 같은 byte 수 | 둘 다 같은 byte 수   |

## 전체 분야별 단위 정리

| 분야           | 단위 기준       | 이유                                                      |
| -------------- | --------------- | --------------------------------------------------------- |
| **RAM**        | 2진법 (GiB)     | 구조적 강제 (이진 주소 체계 + JEDEC 표준 + 경제성)        |
| **네트워크**   | 10진법 (Gbps)   | 통신 역사 + 클럭 주파수 SI 기반 + 물리적 비의존성         |
| **하드디스크** | 10진법 (GB)     | SI 표준 준수 + 물리구조 부합 + 마케팅 이점                |
| **OS 표시**    | 2진법 (GiB)     | 메모리와 일관성 유지 + 프로그래머 관습                    |

## 핵심 정리

1. **1 Gbit = 10억 bit** (10^9) — 네트워크는 SI 단위 (IEEE 802.3, BIPM SI)
2. **25 GbE ÷ 8 = 3.125 GB/s** — 이론적 최대값. 실효 약 95~97% (프로토콜 오버헤드 ~2.5%)
3. **GB와 GiB는 같은 데이터의 다른 표기법** — 3.125 GB = 2.91 GiB = 3,125,000,000 byte
4. **Gbit과 GiB는 동일 차원(정보량)**이지만, bit→byte + SI→2진의 **두 단계 변환**이 필요
5. **혼란 시 항상 byte 수로 환산하라** — 단위는 표기법이지 데이터의 실체가 아니다
6. **네트워크가 10진법인 이유**: 통신 역사 + 클럭 주파수 SI 기반 + 물리적 비의존성 (복합적)
7. **HDD가 10진법인 이유**: SI 표준 준수 + 물리구조 부합 + 마케팅 이점 ("기만"은 아님)
8. **RAM만 2진법인 이유**: 이진 주소 디코더 + JEDEC 표준 + 경제성의 **구조적 강제**
