# Neovim의 버퍼란?

## 현실 비유로 이해하기

> **버퍼 = 책상 위에 펼쳐놓은 종이**
> **디스크 파일 = 서랍 속 원본 문서**

- `:e file.txt` → 서랍에서 원본을 꺼내 **복사본을 책상 위에 펼치는 것**
- 편집 → 책상 위 복사본에 낙서하는 것 (원본은 아직 그대로)
- `:w` → 책상 위 내용을 서랍 원본에 **덮어쓰는 것**
- `:e newfile.txt` → 빈 종이를 책상에 올리고 "newfile.txt"라고 이름표 붙이는 것 (서랍에는 아직 없음)

## 핵심 특징

```
┌─────────────────────────────────────────┐
│           Neovim (책상)                  │
│                                         │
│  버퍼1: init.lua     ← 탭/윈도우로 보임  │
│  버퍼2: lsp.lua      ← 숨겨져 있을 수도  │
│  버퍼3: newfile.txt  ← 아직 디스크에 없음 │
│                                         │
└─────────────────────────────────────────┘
         ↕ :w 로 동기화
┌─────────────────────────────────────────┐
│           디스크 (서랍)                   │
│  init.lua, lsp.lua                      │
│  (newfile.txt는 아직 없음)               │
└─────────────────────────────────────────┘
```

**1. 버퍼 ≠ 파일**

- 파일이 없어도 버퍼는 존재할 수 있다 (`:e newfile.txt`)
- 파일이 있어도 버퍼에 안 열려있을 수 있다

**2. 버퍼 ≠ 탭/윈도우**

- **버퍼**: 메모리에 올라간 텍스트 데이터 (종이)
- **윈도우**: 버퍼를 보여주는 틀 (돋보기/액자) — `Ctrl+w v`로 분할
- **탭**: 윈도우 배치를 묶은 그룹 (책상 자체)

> 하나의 버퍼를 여러 윈도우에서 동시에 볼 수 있다. 같은 종이를 돋보기 2개로 다른 부분을 보는 것처럼!

**3. 숨겨진 버퍼도 살아있음**

- 다른 파일로 이동해도 이전 버퍼는 메모리에 남아있다
- `:ls`로 열린 버퍼 목록을 볼 수 있다

## 자주 쓰는 버퍼 명령어

| 명령    | 의미               | Velox 단축키         |
| ------- | ------------------ | -------------------- |
| `:ls`   | 열린 버퍼 목록     |                      |
| `:bn`   | 다음 버퍼          | `<S-l>`              |
| `:bp`   | 이전 버퍼          | `<S-h>`              |
| `:e`    | 파일을 버퍼에 열기 |                      |
| `:w`    | 버퍼 → 디스크 저장 |                      |
| 버퍼 닫기 | mini.bufremove   | `<leader>x`          |

## 왜 이런 구조일까?

Vim/Neovim이 "버퍼"라는 중간 단계를 두는 이유는 실행 취소(undo), 수정 여부 추적, 여러 윈도우에서 같은 파일 보기 등을 가능하게 하기 위해서다. 디스크를 직접 수정하면 이런 게 어려워진다. 마치 원본 문서에 직접 펜으로 쓰면 지울 수 없지만, 복사본에 쓰면 얼마든지 지우고 다시 쓸 수 있는 것과 같다.
