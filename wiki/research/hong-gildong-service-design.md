> **목적**: 사회 부정부패·세금 탈세 등 부조리를 공개 데이터 기반으로 수집·바이럴·토론하여 사회 개선에 기여하는 한국형 소셜 네트워크.
>
> **본 문서 스코프**: 시장 조사 + 한국 법적 리스크 분석 + 서비스 디자인 + 소프트웨어 아키텍처 + 수익 모델 + Phase 0~3 로드맵. 단일 종합 design doc.
>
> **최초 작성**: 2026-05-16 / **v2 갱신**: 2026-05-16 (document-review 6 페르소나 반영) / **단일 시장**: 한국 / **위험 정책**: 보수적(검증된 공개 데이터 + 출처 강제 + 내부 검증 익명 제보) / **MVP 우선**.

## 0️⃣ Phase 0 — 사용자 검증 및 Kill Criteria (v2 신설)

> document-review의 product-lens·adversarial·feasibility 공통 지적: **"공개 데이터 기반 시민 UGC + 토론" 수요가 한국에 존재하는지 검증되지 않음**. 이를 확인하지 않고 풀스택 MVP에 자원 투입하면 confirmation bias. 따라서 Phase 0을 모든 작업의 선결 조건으로 둔다.

### 0.1 Phase 0 (Pre-MVP, 0-2개월)

| 단계 | 활동 | 검증 가설 |
|------|------|----------|
| **A. 사용자 인터뷰** (2주) | 디시 정치갤·블라인드·뽐뿌 부조리 게시자 20명 + 뉴스타파 후원자 10명 인터뷰 | "출처 강제 + 카테고리화된 부조리 토론" 자발 사용 의향 |
| **B. Wizard-of-Oz 큐레이션** (2-4주) | 운영자 1인이 공개 판결문/감사보고서 50건 큐레이션 + 텀블벅 페이지 | 댓글·후원 의향 정량 |
| **C. 시드 후원 모집** (4-6주) | 텀블벅·해피빈 등 외부 후원 플랫폼에서 사전 후원자 모집 | 100명 시드 후원 달성 가능성 |
| **D. 법무 사전 컨설팅** (병행) | 형법 전문 변호사 1인 1회 자문 + Apple Developer Support 사전 문의 | 핵심 운영 정책 + 앱 심사 리스크 |

### 0.2 Kill Criteria — Phase 0 결과에 따라 STOP / PIVOT 결정

| 지표 | MVP 진입 조건 | 미달 시 |
|------|--------------|---------|
| 사용자 인터뷰 자발 사용 의향 | 30명 中 10명+ "사용하겠다" | STOP — 수요 부재, Pivot 검토 |
| Wizard-of-Oz 후원 전환 (30일) | 시드 후원자 50명+ | PIVOT — 후원 funnel 재설계 |
| 인터뷰 출처 첨부 의향 | 게시 의향자 中 50%+ "출처 첨부 가능" | 정책 재고 — "출처 부재 게시" 별도 라우팅 검토 |
| 형법 전문가 자문 | 중대 차단 이슈 없음 | STOP — 운영 모델 재설계 |

### 0.3 Pivot 옵션 (Phase 0 실패 시)

1. **시민 입법 캠페인 플랫폼**: 법안 발의 추적·압박 — 폭로 대신 정책 변경
2. **부패방지법 신고 가이드 봇**: 권익위·국민신문고 사용성 개선
3. **부조리 데이터 시각화 도구**: 언론사·NGO 대상 B2B (D3 기반)
4. **뉴스타파·참여연대 협업 사이드카**: 자체 플랫폼 대신 기존 단체의 디지털 채널 운영 위탁

### 0.4 후속 검증 단계 KPI (Phase 1 → Phase 2 진입 조건)

| Phase 1 → 2 진입 조건 | 미달 시 |
|---------------------|---------|
| 3개월 후 후원자 100명, 활성 댓글 작성자 200명/월 | 정책·UX 재검토 |
| 6개월 후 후원자 500명, 출처 첨부율 50%+ | STOP 또는 축소 운영 |
| 게시물 reject rate (출처 부재) < 50% | 정책 재설계 |

### 1.3 차별화 포지셔닝

**솔직한 평가** (adversarial 리뷰 반영): 디시·네이트판·블라인드·클리앙은 이미 부조리 폭로+토론+바이럴 기능을 한국에서 수행한다. 홍길동의 진정한 신규 가치는 **2가지뿐**:

1. **출처 강제 + 카테고리 택소노미** — 디시는 ✗, 뉴스타파는 ✗(시민 UGC 없음). 진정한 공백.
2. **검증 신뢰 뱃지 (verified_status)** — 사용자에게 "이 폭로는 어느 단계까지 검증됐는가" 시각화.

다음 3가지는 **운영자에게 가치이지 사용자 needs는 아닐 수 있음** — Phase 0에서 검증 필요:

3. ⚠️ **"제보 → 권익위/언론사 연계" 파이프라인** (안내 링크 수준, I Paid A Bribe와 동일 함정 가능)
4. ⚠️ **정치 중립 헌장** (구조적 폭로 비대칭 문제, §7.1·§7.4 참조)
5. ⚠️ **Impact Report** (후원자 retention 도구이지 신규 acquisition 도구 아님)

🎯 **명시적 위험 인지**: 출처 강제 정책이 폭로 욕구 사용자를 디시·블라인드로 유출시킬 가능성. 출처 reject rate < 50% (§0.4 KPI)를 유지하지 못하면 정책 재설계 필요.

### 2.2 정통망법 70조 (사이버 명예훼손)

- **70조 1항** (사실, 비방 목적): 3년 이하 징역·3,000만 원 벌금, 반의사불벌 (3항) ✓
- **70조 2항** (허위, 비방 목적): 7년 이하 징역·10년 이하 자격정지·5,000만 원 벌금, 반의사불벌 (3항) ✓
- 핵심 요건: **"비방 목적"** vs 공익 목적 (대법원은 주된 동기 기준)
- **익명 허위 주장 차단** 정책은 70조 2항 리스크를 구조적으로 제거

### 2.3 위법성 조각 (형법 310조)

대법원 2023. 2. 2. 선고 2022도13425 판결 기준 4요건:
1. 공연성
2. 진실한 사실
3. **오로지 공공의 이익**
4. 주된 동기가 공익 (부수적 사익 동기가 있더라도 주된 동기가 공익이면 310조 적용)

→ 홍길동 운영 정책으로 명시: **"공인의 공적 직무 비판 + 1차 출처 + 비방 표현 금지"** 3원칙 게시물 자동 검사.

### 2.4 플랫폼 운영자 책임 — 세이프하버 활용

**대법원 2009. 4. 16. 선고 2008다53812 전원합의체 판결** (v2까지 인용했던 "2007다8333"은 사건 번호 오류 — v3 정정) 삭제 의무 3요건:
1. 피해자가 게시물을 구체적·개별적으로 특정하여 삭제 요청
2. 게시물의 명예훼손 불법성이 **명백히 인식**됨
3. 삭제 조치가 기술적·경제적으로 가능

→ **3요건 미충족 시 부작위 책임 없음**. 이를 위해:
- 임시조치 신고 폼 표준화
- §7.4 SLA + immutable audit log
- "명백 불법" 판단 기준 내부 문서화

> ⚠️ **추가 인용 주의 (rl-verify 정정)**: 대법원 2022다284513(2024.1.4.) 판결은 v2에서 "인터넷 운영자 명예훼손 책임"의 근거로 인용했으나, 실제 판결 내용은 국정원 대변인의 "종북 사이트" 발언이 운영자에 대한 명예훼손에 **해당하지 않는다**는 방향. 운영자 책임 법리 인용 시 2008다53812(2009)를 기본으로 사용하고, 2022다284513은 보조 참고로 한정.

### 2.5 개인정보보호법 (2023 개정)

### 3.2 데이터 모델 (ICIJ 4노드 → 한국 변환)

```
[지식 그래프]
(:공직자 {직책, 임기, 소속기관})
  -[:소유·대표·관계]→
(:기업·단체 {법인번호, 업종, 설립일})
  -[:계약·수주]→
(:사건 {카테고리, 발생일, 피해금액, 상태, verified_status})
  -[:출처]→
(:자료 {URL, 문서유형, 검증여부, 접근일})
```

**다축 인덱싱**: 시간(`incident_date`) × 지역(`jurisdiction_code` ISO 3166-2 KR-XX) × 금액(`amount_krw`) × 관련인물(그래프 외래키) × 유형(`categories[]`) × 검증도(`verified_status`).

#### 검증 상태 ↔ UI 뱃지 매핑 (v2 신설, coherence 리뷰 반영)

데이터 모델 `verified_status`와 UI 뱃지의 매핑을 명확화:

| `verified_status` (DB) | UI 뱃지 | 의미 | 신고 파이프라인 (§3.5) 자동 안내 |
|----------------------|---------|------|--------------------------------|
| `unverified` | 🔴 미검증 | 제보자 단일 출처, 검증 안 함 | ✗ |
| `crowd_verified` | 🟡 커뮤니티 검증 | 2인 이상 이용자 "맞다" 투표 + 출처 URL | ✗ |
| `source_linked` | 🟡 커뮤니티 검증 | 공개 데이터(뉴스·공시·판례) 직접 링크 | △ (선택적) |
| `officially_confirmed` | 🟢 공식 확인 | 권익위·법원·언론사 공식 보도 확인 | ✅ |

#### 그래프 depth 제한 (v2 보강, feasibility 리뷰 반영)

MVP는 `depth < 3`으로 시작하지만, **한국 재벌 구조는 평균 4-6 hops, 최대 7-9단계**(공정위 2024 발표). 따라서:
- **Phase 1**: depth < 3 (단순 관계 표시), `case_entities[]` 배열 외래키 우선 사용 (entity_links 보조)
- **Phase 2** (데이터 1,000건+): entity_links + recursive CTE depth 5-7 확장
- **Phase 3**: materialized path 또는 Apache AGE (PostgreSQL graph extension) 검토

### 4.3 데이터 모델 (PostgreSQL + recursive CTE)

```sql
abuse_cases  (id, title, category[], status, severity_score, verified_status, created_at)
entities     (id, name, type ENUM[person, org, region, incident], aliases jsonb)
entity_links (source_id, target_id, relation_type, weight, source_case_id)
evidences    (id, case_id, type, r2_key, encrypted, hash_sha256)
tips         (id, case_id, content_encrypted BYTEA, source_ip_hash, status)
mod_queue    (id, content_id, ai_score, ai_flags jsonb, human_status, legal_hold)
legal_log    (id BIGSERIAL, action_type, target_id, requester, reason,
              response_at, prev_hash)  -- 해시 체인 (RFC 3161 타임스탬프)
```

**그래프 쿼리** (Neo4j 대신 PostgreSQL CTE — 2-3 depth는 ms 단위 충분):
```sql
WITH RECURSIVE graph AS (
  SELECT target_id, 1 AS depth FROM entity_links WHERE source_id = $1
  UNION ALL
  SELECT el.target_id, g.depth + 1
  FROM entity_links el JOIN graph g ON el.source_id = g.target_id
  WHERE g.depth < 3
)
SELECT DISTINCT e.* FROM entities e JOIN graph g ON e.id = g.target_id;
```

| Resend (트랜잭션 메일) | Free tier | $0 |
| **합계 (인프라만)** | | **~$70-95/월** (v2 추정 $87-100 → v3 정정) |

> **R2 egress (v3 정정)**: Cloudflare R2의 **egress는 전방향 무료** (인터넷·다른 클라우드 모두). v2까지 명시했던 "외부 클라우드 egress $0.09/GB"는 오류 — AWS S3 단가를 잘못 적용한 것. Phase 3 NCP 마이그레이션 시 발생하는 비용은 R2 egress가 아니라 (a) Postgres 덤프·복원 다운타임, (b) DNS 전파, (c) Meilisearch 재색인, (d) Tor onion 주소 변경 등 운영 비용 측면 검토 필요.

> **Fly.io 가격 (v3 정정)**: 초당 과금 구조. shared-cpu-1x 2GB ~$11/월, 1GB ~$6/월. 숨은 비용: dedicated IPv4 +$2/월, 볼륨 스토리지, 외부 egress $0.02/GB. v2까지 추정한 "$20/$10"은 과대 추정.
