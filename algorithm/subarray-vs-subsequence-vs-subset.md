좋은 질문입니다! 핵심은 **"subarray"는 연속된 부분 배열**이라는 조건입니다.

## 양수만 골라 더하면 안 되는 이유

```
인덱스:  0   1   2   3   4   5   6   7   8
배열:   [-2,  1, -3,  4, -1,  2,  1, -5,  4]
```

양수만 골라 더하면 `1 + 4 + 2 + 1 + 4 = 12`로 훨씬 크죠. 하지만 이건 인덱스 `1, 3, 5, 6, 8`을 **띄엄띄엄** 골라야 합니다. 중간에 `-3`, `-1`, `-5`를 건너뛰는 거죠.

**subarray는 중간을 건너뛸 수 없습니다.** 반드시 연속된 구간이어야 합니다.

## subarray vs subsequence vs subset

| 개념          | 조건                   | 예시 (위 배열에서)                     |
| ------------- | ---------------------- | -------------------------------------- |
| **subarray**  | 연속된 구간            | `[4, -1, 2, 1]` (인덱스 3~6)          |
| subsequence   | 순서 유지, 불연속 가능 | `[1, 4, 2, 1, 4]` (인덱스 1,3,5,6,8)  |
| subset        | 아무거나 선택          | `{1, 2, 4, 4}`                         |

## 현실 비유

기차 칸이라고 생각하면 됩니다. 10칸짜리 기차에서 **연결된 칸들만** 고를 수 있어요. "3번 칸부터 6번 칸까지" 이런 식으로요. 3번, 5번, 8번 칸만 따로 떼어서 연결할 수는 없습니다.

그래서 `[4, -1, 2, 1]`이 답인 이유는, **연속 구간 중에서** 합이 `6`으로 가장 크기 때문입니다. `-1`이 끼어 있지만, 그걸 포함해야 `4`와 `2, 1`을 연결할 수 있으니 감수하는 거죠.
