# 비디오 Super-Resolution 딥러닝 모델 심층 분석

## 배경: 문서에서 언급된 기술

`career/inshorts-ai-platform-lead-analysis.md`에서 언급된 핵심 기술:

- **핵심 기술**: CVPR 2023 SOTA 비디오 초해상화(Super-Resolution) 모델 기반
- **CAIO 오지형 교수**: CVPR에서 Google DeepMind, Adobe, Netflix 연구진과 비디오 슈퍼레졸루션 + 프레임 보간 SOTA 모델 워크숍 발표
- **운영 방식**: 온프레미스 GPU 인프라에서 AI 추론 수행

```python
video = receive_upload("movie_1080p.mp4")        # 고객 업로드
frames = ffmpeg.extract_frames(video)             # 프레임 분할
for batch in chunk(frames, batch_size=16):
    upscaled = model.inference(batch)             # GPU에서 추론 ⭐
    save(upscaled)
result = ffmpeg.merge_frames(upscaled_frames, audio)  # 영상 합성
```

이 포지션이 고민하는 것은 `model.inference(batch)` 한 줄이 아니라, **그 주변의 모든 것**: 프레임을 GPU 메모리에 효율적으로 올리는 법, batch_size 최적화, 수천 개 동시 요청 스케줄링, GPU 장애 복구, 고객 간 데이터 격리.

## ML인가 DL인가?

### 결론: 확실하게 DL (딥러닝)

| 구분     | ML (기계학습)                        | DL (딥러닝)                              |
| -------- | ------------------------------------ | ---------------------------------------- |
| 비유     | 레시피를 보고 요리하는 것            | 수천 번 먹어보고 스스로 레시피를 만드는 것 |
| 특징추출 | 사람이 "이런 특징을 봐" 알려줌       | 모델이 스스로 특징을 발견                 |
| 대표 예  | SVM, Random Forest, Linear Regression | CNN, GAN, Transformer, Diffusion         |

**왜 DL인가?**
- 저해상도 픽셀에서 고해상도 픽셀을 **"생성"**해야 함 → 단순 규칙이 아니라 수백만 개의 학습된 패턴이 필요
- CVPR (컴퓨터 비전 최고 학회)에서 SOTA라는 것 자체가 딥러닝 기반이라는 뜻

## 오픈소스 모델 현황

HuggingFace에 "super-resolution" 태그로 **193개 이상**의 모델이 등록되어 있다. 다만 대부분은 **이미지** SR이고, **비디오** SR은 주로 GitHub에 공개되어 있다.

### 주요 오픈소스 비디오 SR 모델

| 모델           | 위치                                                                  | 특징                                  |
| -------------- | --------------------------------------------------------------------- | ------------------------------------- |
| **BasicVSR++** | [GitHub](https://github.com/ckkelvinchan/BasicVSR_PlusPlus)           | CVPR 2022. 양방향 재귀 + Flow 정렬    |
| **RealBasicVSR** | [GitHub](https://github.com/ckkelvinchan/RealBasicVSR)              | 실제 영상(노이즈 포함)에 강한 버전    |
| **RVRT**       | [GitHub](https://github.com/JingyunLiang/RVRT)                       | NeurIPS 2022. Recurrent + Transformer |
| **VRT**        | [GitHub](https://github.com/JingyunLiang/VRT)                        | Video Restoration Transformer         |
| **Swin2SR**    | [HuggingFace](https://huggingface.co/docs/transformers/model_doc/swin2sr) | HuggingFace Transformers 공식 통합    |

> 🍳 요리 레시피(모델 코드)와 완성된 반조리 식품(pretrained weights)이 모두 공개되어 있다. 누구나 다운받아서 바로 "데워 먹을" 수 있는 상태.

### GPU 없이 돌릴 수 있을까?

**이론적으로는 가능하지만, 실질적으로 불가능에 가까움.**

| 환경                  | 480p→1080p 60분 영상 기준                    | 비유               |
| --------------------- | -------------------------------------------- | ------------------ |
| 🖥️ CPU만              | **수시간~수십 시간** (모델/CPU에 따라 크게 상이) | 자전거로 서울→부산 |
| 🎮 GPU (RTX 3090)     | **~30분~1시간**                               | 자동차로 서울→부산 |
| ⚡ 전문 GPU (A100)     | **~10~20분**                                  | KTX로 서울→부산    |

**왜 GPU가 필수인가?**
- SR 모델은 **모든 픽셀을 동시에 계산**해야 함 → 병렬 연산이 핵심
- GPU는 수천 개의 코어가 동시에 일하는 구조 (CPU는 수십 개)
- Intel OpenVINO나 FPGA 같은 대안도 있지만, GPU 대비 성능이 크게 떨어짐

## SOTA 알고리즘 계보

SOTA = "State Of The Art" (현재 최고 성능). 100미터 달리기 세계기록이 계속 깨지는 것처럼 시대별로 바뀌어 왔다:

```
2021 ─── BasicVSR (단방향 재귀) [CVPR 2021]
  │      "앞에서 뒤로만 프레임을 읽음"
  ▼
2022 ─── BasicVSR++ (양방향 재귀 + Flow 정렬) [CVPR 2022] ⭐ 현재도 강력한 baseline
  │      "앞뒤 양쪽에서 프레임을 읽음"
  │
  ├── VRT (Video Restoration Transformer) [IEEE TIP / arXiv 2022]
  │      "Transformer로 시공간 동시 처리"
  │
  ├── RVRT (Recurrent VRT) [NeurIPS 2022] ⭐
  │      "VRT의 효율 개선 버전 — 메모리↓ 성능↑"
  ▼
2024 ─── Diffusion 기반 SR (SinSR, StableSR) [CVPR 2024]
  │      "확산 모델로 더 자연스러운 디테일 생성"
  │
  ├── SparkVSR (Sparse Keyframe Propagation)
  │      "모든 프레임이 아니라 핵심 프레임만 처리 → 속도↑"
  ▼
2025 ─── MamEVSR (Mamba + Event Camera) [CVPR 2025]
  │      "State Space Model로 선형 복잡도 달성"
  │
  ├── Efficient VSR for Real-time Rendering
         "게임/렌더링용 실시간 SR"
```

### 현재(2025) SOTA 경쟁 구도

| 접근법                  | 대표 모델        | 장점                       | 단점                     |
| ----------------------- | ---------------- | -------------------------- | ------------------------ |
| **Recurrent + Flow**    | BasicVSR++       | 안정적, 빠름, 검증됨       | 디테일 생성 한계         |
| **Transformer**         | RVRT, VRT        | 높은 품질, 장거리 의존성    | GPU 메모리 많이 필요     |
| **Diffusion**           | StableSR, SinSR  | 가장 자연스러운 디테일      | 느림, 할루시네이션 위험  |
| **State Space (Mamba)** | MamEVSR          | 선형 복잡도, 효율적         | 아직 초기 단계           |

💡 "SOTA 하나"가 아니라 **용도별로 다른 SOTA**가 존재한다. 상용 제품은 주로 BasicVSR++/RVRT 계열(안정성+속도)을, 연구에서는 Diffusion/Mamba 계열을 추구한다.

## 핵심 아키텍처: Recurrent + Attention 기반 딥러닝

### 🔁 Recurrent (재귀) = "기억하면서 순서대로"

> 📚 소설책을 읽을 때, 앞 장의 내용을 **기억하면서** 다음 장을 읽는 것

```
프레임1 → [처리] → 특징1 ─┐
                          ↓ (기억 전달)
프레임2 → [처리 + 기억] → 특징2 ─┐
                                ↓ (기억 전달)
프레임3 → [처리 + 기억] → 특징3
```

- 비디오는 **순서가 있는 프레임들의 연속**
- 프레임 3을 처리할 때, 프레임 1, 2에서 본 정보를 **기억**으로 넘겨받음
- 비유: 영화를 볼 때 "아까 그 장면에서 저 사람 얼굴이 이랬으니까..." 하고 떠올리는 것

### 👀 Attention (어텐션) = "중요한 것에 집중"

> 🏫 교실에서 선생님이 중요한 부분에 형광펜을 치는 것

```
프레임의 모든 픽셀 중에서:
  [나무] [하늘] [사람 얼굴] [바닥] [건물]
          ↓ Attention 가중치
  [0.05] [0.02] [0.80] ⭐  [0.03] [0.10]

→ "사람 얼굴"에 80% 집중해서 처리!
```

- 모든 정보를 균일하게 보지 않고, **중요한 부분에 더 많은 연산력을 투입**
- 비유: 시험 공부할 때 전체를 골고루 읽는 게 아니라, 중요한 부분에 밑줄 치고 집중하는 것

### 🔁 + 👀 합치면? = RVRT 같은 모델

```
          시간 방향 (Recurrent)
          ───────────────────→
프레임1   프레임2   프레임3   프레임4
  │         │         │         │
  ▼         ▼         ▼         ▼
[특징추출] [특징추출] [특징추출] [특징추출]
  │         │         │         │
  └────→ [Attention: 프레임1과 2 중 어디가 중요?]
            │         │
            └────→ [Attention: 2와 3 중 어디가 중요?]
                      │         │
                      └────→ [Attention: 3과 4 중 어디가 중요?]
                                │
                                ▼
                          [고해상도 출력]
```

| 역할         | Recurrent만          | Attention만          | 합친 것                      |
| ------------ | -------------------- | -------------------- | ---------------------------- |
| 시간 정보    | ✅ 순서대로 기억      | ❌ 순서 무관          | ✅ 순서 + 중요도              |
| 공간 정보    | ❌ 전체 균일 처리     | ✅ 중요 부분 집중     | ✅ 시공간 동시 처리           |
| 메모리 효율  | ✅ 적음              | ❌ 많음               | ⚠️ 중간                      |
| 비유         | 소설을 순서대로 읽기 | 핵심만 골라 읽기      | **순서대로 읽되 핵심에 형광펜** |

## Super-Resolution vs Frame Interpolation

### Super-Resolution: 해상도를 높인다 = 픽셀을 작게 많이 만든다

```
저해상도 (4x4 = 16픽셀)        고해상도 (8x8 = 64픽셀)
┌──┬──┬──┬──┐                ┌─┬─┬─┬─┬─┬─┬─┬─┐
│  │  │  │  │                │ │ │ │ │ │ │ │ │
├──┼──┼──┼──┤     SR →       ├─┼─┼─┼─┼─┼─┼─┼─┤
│  │  │  │  │                │ │ │ │ │ │ │ │ │
├──┼──┼──┼──┤                ├─┼─┼─┼─┼─┼─┼─┼─┤
│  │  │  │  │                │ │ │ │ │ │ │ │ │  ← 새로 생긴 픽셀들!
├──┼──┼──┼──┤                ├─┼─┼─┼─┼─┼─┼─┼─┤
│  │  │  │  │                │ │ │ │ │ │ │ │ │
└──┴──┴──┴──┘                └─┴─┴─┴─┴─┴─┴─┴─┘
```

핵심은 빈 칸을 **어떻게** 채우느냐:

| 방법              | 원리                         | 비유                                 | 결과        |
| ----------------- | ---------------------------- | ------------------------------------ | ----------- |
| **보간법**        | 주변 픽셀의 평균값            | 물감을 물에 타서 흐리게               | ❌ 흐릿함   |
| **딥러닝 SR**     | 학습된 패턴으로 디테일 생성   | 화가가 빠진 부분을 **상상해서** 그림  | ✅ 선명함   |

> 🖼️ 4x4 모자이크 그림이 있다. 단순 확대하면 네모 블록이 커질 뿐이다(보간법). 하지만 화가(딥러닝 모델)가 "이 네모는 코 부분이니까 이런 디테일이 있을 거야"라고 **추론해서 그려넣는 것**이 SR이다.

#### Sub-pixel Convolution (전통적 CNN 기반 SR에서 가장 많이 쓰이는 업스케일링 기법)

```
모델 내부에서 채널 수를 늘림 → 채널들을 공간으로 재배치

[1채널, H, W]  →  [4채널, H, W]  →  [1채널, 2H, 2W]
 (저해상도)      (정보 풍부하게)     (고해상도로 펼침!)
```

⚠️ SR은 단순히 픽셀을 "쪼개는" 게 아니라, **없던 정보를 학습된 패턴으로 "생성"**하는 것. 그래서 DL이 필수.

### Frame Interpolation: 프레임 수를 늘린다

SR과 FI는 **서로 직교하는 개념**:

```
            공간 (해상도)
            ↑
            │   ┌──────────────────┐
  4K ───────│───│  SR + FI 동시적용  │ ← 최종 목표
            │   └──────────────────┘
            │          ↗        ↑
  1080p ────│── [원본] ──────── [FI만 적용]
            │     │
   720p ────│── [SR만 적용]
            │
            └──────────────────────→ 시간 (프레임 수)
                24fps    60fps
```

| 구분           | Super-Resolution       | Frame Interpolation       |
| -------------- | ---------------------- | ------------------------- |
| **뭘 늘리나**  | 픽셀 수 (공간)         | 프레임 수 (시간)           |
| **비유**       | 사진을 더 선명하게     | 플립북 페이지를 추가       |
| **입력→출력**  | 720p → 4K              | 24fps → 60fps              |
| **채우는 것**  | 픽셀 사이의 디테일     | 프레임 사이의 움직임       |

```
시간 →

원본 (24fps):
프레임1        프레임2        프레임3
[🚶 사람 왼쪽]              [🚶 사람 오른쪽]
     ↓                          ↓
보간 후 (48fps):
프레임1   [새 프레임]  프레임2  [새 프레임]  프레임3
[🚶 왼쪽] [🚶 중간!]  [🚶 오른] [🚶 더오른!]
```

> 📖 만화책에 1페이지: 캐릭터가 왼쪽, 3페이지: 캐릭터가 오른쪽. 프레임 보간은 **2페이지에 캐릭터가 가운데 있는 그림을 AI가 그려넣는 것**.

## 학습 과정: 모델이 어떻게 만들어지는가

```
학습 데이터: 고해상도 영상 (4K 원본)
    ↓ 인위적으로 해상도를 낮춤 (degradation)
저해상도 영상 (1080p, 720p 등)
    ↓
모델에게 "이 저해상도 → 저 고해상도" 쌍을 수백만 번 보여줌
    ↓
모델이 "저해상도에서 고해상도를 복원하는 패턴"을 학습
```

**Loss 함수** (모델이 얼마나 잘했는지 측정하는 기준):
- **L1/L2 Loss**: 픽셀 단위로 원본과 얼마나 비슷한가
- **Perceptual Loss**: 사람 눈으로 봤을 때 얼마나 자연스러운가
- **GAN Loss** (일부 모델): 생성된 이미지가 진짜처럼 보이는가

## 핵심 구성 요소: 비디오 SR 파이프라인

```
1. Optical Flow Estimation (광학 흐름)
   → 프레임 간 움직임을 추적 (어떤 픽셀이 어디로 이동했나?)
   → 비유: 플립북에서 캐릭터가 어디로 움직이는지 추적하는 것

2. Feature Alignment (특징 정렬)
   → 여러 프레임의 같은 물체를 정확히 맞춤
   → 비유: 여러 장의 흐린 사진을 겹쳐서 선명하게 만드는 것

3. Aggregation (특징 결합)
   → 정렬된 프레임들의 정보를 합침
   → 비유: 퍼즐 조각들을 모아 완성된 그림을 만드는 것

4. Upsampling (해상도 증가)
   → Sub-pixel convolution 등으로 실제 해상도를 높임
   → 비유: 작은 레고 블록을 더 정밀한 블록으로 교체하는 것
```

SR과 Frame Interpolation이 같이 발표되는 이유:
1. **기술적으로 공유하는 것이 많음**: 둘 다 Optical Flow(움직임 추적)를 사용하는 경우가 많음
2. **상업적으로 시너지**: "옛날 SD 영화를 4K 60fps로" → 한 번에 두 가지를 해결
3. **같은 GPU 인프라**를 재활용 가능

## 핵심 정리

1. 비디오 Super-Resolution은 **확실히 DL(딥러닝)** — ML로는 구현 불가능한 수준의 패턴 학습이 필요
2. 오픈소스 모델이 **HuggingFace/GitHub에 다수 존재**하며, pretrained weights도 공개되어 바로 사용 가능
3. **GPU 없이는 실용적 사용 불가** — CPU로는 수시간~수십 시간 소요
4. SOTA는 하나가 아니라 **용도별로 다름**: 상용(BasicVSR++/RVRT) vs 연구(Diffusion/Mamba)
5. 핵심 아키텍처 = **Recurrent(시간 순서 기억) + Attention(중요 부분 집중)**
