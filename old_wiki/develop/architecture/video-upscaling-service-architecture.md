# Video Upscaling Service Architecture

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                        클라이언트                             │
└────────────────────────────┬────────────────────────────────┘
                             ↓
┌────────────────────────────────────────────────────────────┐
│                     L7 Load Balancer                        │
└────────────────────────────┬───────────────────────────────┘
                             ↓
┌────────────────────────────────────────────────────────────┐
│                   API Gateway ×N                            │
│               (인증, 레이트리밋)                              │
└────────────────────────────┬───────────────────────────────┘
                             ↓
┌────────────────────────────────────────────────────────────┐
│                   API Server ×N                             │
│                     (FastAPI)                                │
│                                                             │
│  • 비디오 → S3 업로드                                        │
│  • S3 → EFS 복사 트리거                                      │
│  • FFprobe로 메타데이터만 읽음 (디코딩 안 함)                  │
│  • 청크 분할 계획 (산수만)                                    │
│  • DB에 Job + Chunk 레코드 저장 (숫자만)                      │
│  • Celery chord로 청크별 Task 발행                           │
│  • job_id 즉시 반환                                         │
│  • 상태 조회 API 제공 (/jobs/{id}/status)                    │
└──────┬──────────────────┬──────────────────┬───────────────┘
       ↓                  ↓                  ↓
┌──────────────┐  ┌───────────────┐  ┌──────────────────────┐
│  PostgreSQL   │  │ Redis / Kafka  │  │        S3            │
│               │  │  (Job Queue)   │  │   (영구 저장소)       │
│ • Job 테이블  │  │               │  │  inputs/             │
│ • Chunk 테이블│  │ • 청크별 Task │  │  chunks/             │
│ • 상태/진행률 │  │ • chord 콜백  │  │  outputs/            │
└──────────────┘  └───────┬───────┘  └──────────┬───────────┘
                          ↓                      ↓ 1회 복사
═══════════════════════════════════════════════════════════════
              Tier 1: CPU 서버 클러스터
═══════════════════════════════════════════════════════════════

┌──────────────────────────────────────────────────────────┐
│                 EFS / NFS (공유 파일시스템)                 │
│                                                          │
│  /mnt/efs/videos/{job_id}/video.mp4  ← S3에서 1회 복사    │
│                                                          │
│  • 모든 Worker가 마운트하여 읽기 전용 접근                   │
│  • Worker별 다운로드 불필요                                 │
│  • FFmpeg -ss seek으로 각자 원하는 구간만 디코딩             │
│  • Job 완료 후 정리                                       │
└──────────────────────────────────────────────────────────┘
                          ↑ 마운트 (읽기 전용)
┌──────────────────────────────────────────────────────────┐
│              Celery Worker Pool ×N                        │
│                                                          │
│  각 Worker가 독립적으로 1개 청크 처리:                      │
│                                                          │
│  1. EFS에서 비디오 직접 읽기 (다운로드 불필요)               │
│  2. FFmpeg -ss seek → 해당 프레임 구간만 디코딩             │
│  3. 프레임 → 512×512 타일 분할                             │
│  4. 배치 단위로 A100 TIS 호출 (gRPC)                       │
│  5. 업스케일된 타일 수신 → 프레임 재조합                     │
│  6. T4 TIS에 후처리 + 인코딩 요청 (gRPC)                   │
│  7. chunk_XXX.mp4 → S3 업로드                             │
│  8. DB 체크포인트 기록                                     │
│                                                          │
│  실패 시: 해당 청크만 자동 재시도 (max 3회)                  │
└──────────┬──────────────────────────────┬────────────────┘
           ↓ gRPC                         ↓ gRPC
┌─────────────────────┐       ┌─────────────────────────┐
│   L7 LB (A100용)    │       │    L7 LB (T4용)         │
└──────────┬──────────┘       └──────────┬──────────────┘
           ↓                              ↓
═══════════════════════        ═══════════════════════════
 Tier 2: A100 GPU 서버          Tier 3: T4/L4 GPU 서버
   (추론 전담)                    (후처리 + 인코딩)
═══════════════════════        ═══════════════════════════
┌─────────────────────┐       ┌─────────────────────────┐
│  TIS ×N대            │       │  TIS ×N대                │
│  (각 A100 ×8)        │       │  (각 T4/L4 ×1)          │
│                      │       │                          │
│  model_repository/   │       │  model_repository/       │
│  ├── preprocessor    │       │  ├── color_correct (GPU) │
│  │   (CPU, 정규화)   │       │  ├── qa_model (GPU)      │
│  ├── sr_model        │       │  ├── watermark (CPU)     │
│  │   (GPU, FP16)     │       │  ├── encoder             │
│  ├── postprocessor   │       │  │   (CPU, NVENC 호출)   │
│  │   (CPU, 역정규화) │       │  └── post_pipeline (BLS) │
│  └── pipeline (BLS)  │       │                          │
│                      │       │  CUDA Cores: 후처리 모델  │
│  동적 배치 [4, 8]    │       │  NVENC HW: 인코딩 (병렬)  │
│  GPU 활용률 70~85%   │       └─────────────────────────┘
└─────────────────────┘
           ↓                              ↓
═══════════════════════════════════════════════════════════
                      S3 / Object Storage
═══════════════════════════════════════════════════════════
┌─────────────────────────────────────────────────────────┐
│  inputs/{job_id}/video.mp4        ← 원본                 │
│  chunks/{job_id}/chunk_000.mp4    ← 청크 (임시)          │
│  chunks/{job_id}/chunk_001.mp4                           │
│  ...                                                     │
│  outputs/{job_id}/result.mp4      ← 최종 결과             │
└─────────────────────────────────────────────────────────┘
```

## Sequence Diagrams

### 1. Job 생성 ~ Task 발행

```
Client        LB/GW      API Server       DB        Redis       S3         EFS
  │             │             │            │          │          │           │
  │──비디오 업로드→            │            │          │          │           │
  │             │──라우팅─────→│            │          │          │           │
  │             │             │──비디오 저장──────────────────────→│          │
  │             │             │                       │          │           │
  │             │             │──S3→EFS 복사 트리거────────────────────────→│
  │             │             │                       │          │  video.mp4│
  │             │             │                       │          │           │
  │             │             │──FFprobe (메타데이터)──────────────────────→│
  │             │             │←─fps:30, frames:216000───────────────────│
  │             │             │                       │          │           │
  │             │             │  산수: 216000/300 = 720청크                  │
  │             │             │                       │          │           │
  │             │             │──Job + 720 Chunk 레코드→│         │           │
  │             │             │                       │          │           │
  │             │             │──chord(720 Tasks)─────→│         │           │
  │             │             │  process_chunk(0, 0~299)│         │           │
  │             │             │  process_chunk(1, 300~599)        │           │
  │             │             │  ...                   │         │           │
  │             │             │  process_chunk(719, ...)│         │           │
  │             │             │                       │          │           │
  │←──job_id 반환←────────────│                       │          │           │
  │             │             │                       │          │           │
```

### 2. 청크 처리 (Worker 1개 기준)

```
Worker         EFS         A100 TIS (LB)      T4 TIS (LB)       S3        DB
  │             │               │                  │              │         │
  │──seek(0s)──→│               │                  │              │         │
  │  FFmpeg 디코딩               │                  │              │         │
  │  (프레임 0~299만)            │                  │              │         │
  │  ※ 다운로드 없이 직접 읽기    │                  │              │         │
  │             │               │                  │              │         │
  │  타일 분할                   │                  │              │         │
  │  (300프레임 × 6타일          │                  │              │         │
  │   = 1800타일)                │                  │              │         │
  │             │               │                  │              │         │
  │  ┌─ 배치 루프 (32프레임분씩) ────────────────────────────────┐│         │
  │  │                          │                  │             ││         │
  │  │──192타일 배치 gRPC───────→│                  │             ││         │
  │  │                          │  preprocessor    │             ││         │
  │  │                          │  → sr_model(GPU) │             ││         │
  │  │                          │  → postprocessor │             ││         │
  │  │←─업스케일된 타일─────────│                  │             ││         │
  │  │                          │                  │             ││         │
  │  │  타일 재조합 → 프레임     │                  │             ││         │
  │  │                          │                  │             ││         │
  │  └─ 배치 루프 반복 ──────────────────────────────────────────┘│         │
  │             │               │                  │              │         │
  │  전체 업스케일된 프레임 완성   │                  │              │         │
  │             │               │                  │              │         │
  │──프레임 gRPC─────────────────────────────────→│              │         │
  │             │               │                  │  색보정(GPU) │         │
  │             │               │                  │  QA(GPU)    │         │
  │             │               │                  │  워터마크    │         │
  │             │               │                  │  NVENC인코딩 │         │
  │←─chunk_000.mp4───────────────────────────────│              │         │
  │             │               │                  │              │         │
  │──chunk S3 업로드──────────────────────────────────────────→│         │
  │             │               │                  │              │         │
  │──체크포인트 기록────────────────────────────────────────────────────→│
  │  (chunk_0: COMPLETED)       │                  │              │         │
  │             │               │                  │              │         │
```

### 3. 전체 병렬 처리 → Finalize

```
Redis          EFS           Worker1        Worker2        Worker3        Worker4
  │             │               │              │              │              │
  │             │  video.mp4    │              │              │              │
  │             │  (1회 복사됨)  │              │              │              │
  │             │               │              │              │              │
  │──chunk 0───────────────────→│              │              │              │
  │──chunk 1───────────────────────────────→│              │              │
  │──chunk 2─────────────────────────────────────────────→│              │
  │──chunk 3───────────────────────────────────────────────────────────→│
  │             │               │              │              │              │
  │             │←─seek(0s)────│              │              │              │
  │             │←─seek(10s)───────────────│              │              │
  │             │←─seek(20s)─────────────────────────────│              │
  │             │←─seek(30s)───────────────────────────────────────────│
  │             │               │              │              │              │
  │             │  ※ 같은 파일을 여러 Worker가 동시에 읽기 전용 접근         │
  │             │  ※ 각자 다른 위치를 seek → 충돌 없음                      │
  │             │               │              │              │              │
  │  [디코딩→추론→인코딩]  [디코딩→추론→인코딩]  [동시 진행]   [동시 진행]      │
  │             │               │              │              │              │
  │←─chunk 0 완료──────────────│              │              │              │
  │──chunk 4───────────────────→│              │              │              │
  │             │               │              │              │              │
  │←─chunk 1 완료──────────────────────────│              │              │
  │──chunk 5───────────────────────────────→│              │              │
  │             │               │              │              │              │
  │  ... 720개 청크가 4개 Worker에 분배되어 병렬 처리 ...                      │
  │             │               │              │              │              │
  │←─chunk 719 완료 (마지막)     │              │              │              │
  │             │               │              │              │              │
  │  chord 감지: 720개 전부 완료                                              │
  │             │               │              │              │              │
  ↓             │               │              │              │              │
Finalize Worker │               │              │              │              │
  │             │               │              │              │              │
  │  S3에서 chunk_000 ~ chunk_719 다운로드                                    │
  │  FFmpeg concat (재인코딩 없이 이어붙이기)                                   │
  │  원본 오디오 합성                                                          │
  │  final.mp4 → S3 업로드                                                   │
  │  EFS 임시 파일 삭제 (video.mp4)                                           │
  │  S3 임시 청크 파일 삭제                                                    │
  │  DB Job 상태 → COMPLETED                                                 │
  │  콜백 알림                                                                │
```
