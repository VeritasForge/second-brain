## 32비트 정수의 최대값은 왜 2^31 - 1일까?

### 32비트 = 스위치 32개

32비트란 0과 1을 넣을 수 있는 **칸이 32개**라는 뜻이야. 전등 스위치 32개가 나란히 있다고 상상해봐.

각 스위치는 켜짐(1) 또는 꺼짐(0), 두 가지 상태를 가지니까 **총 조합 수 = 2^32 = 4,294,967,296가지**야.

### 왜 최대값에 -1을 하는 걸까?

스위치 **2개**로 먼저 생각해보자:

```
OFF OFF = 00 = 0
OFF ON  = 01 = 1
ON  OFF = 10 = 2
ON  ON  = 11 = 3
```

조합은 **4가지**(= 2^2)인데, **0부터 세기 시작**하니까 최대값은 4가 아니라 **3** = 2^2 - 1

손가락으로 생각해봐. 손가락 10개로 0부터 세면 0~9까지, 최대 **9** = 10 - 1이잖아. 10개의 숫자를 표현하지만 최대값은 10이 아니라 9인 것!

같은 원리로 확장하면:

```
스위치 2개  → 조합 2^2개  → 최대값 2^2 - 1 = 3
스위치 8개  → 조합 2^8개  → 최대값 2^8 - 1 = 255
스위치 32개 → 조합 2^32개 → 최대값 2^32 - 1 = 4,294,967,295
```

항상 **"0부터 세니까 -1"**이 핵심이야.

### 그런데 왜 2^32 - 1이 아니라 2^31 - 1?

위의 `2^32 - 1`은 **부호 없는(unsigned)** 정수, 즉 양수만 쓸 때의 이야기야.

**음수도 표현**해야 하면? 맨 앞 1칸을 **부호 표시용**으로 빌려줘야 해:

```
[부호][숫자 31칸]
  0    000...000  = +0
  0    111...111  = +2^31 - 1  (양수 최대)
  1    000...000  = -2^31      (음수 최소)
  1    111...111  = -1
```

- 부호 비트: `0` = 양수, `1` = 음수
- 숫자용 칸: **31칸** → 양수 범위 = `0 ~ 2^31 - 1`

### 정리

| 종류            | 사용 비트             | 범위              |
| --------------- | --------------------- | ----------------- |
| unsigned 32비트 | 32칸 전부             | 0 ~ 2^32 - 1     |
| signed 32비트   | 1칸 부호 + 31칸 숫자  | -2^31 ~ 2^31 - 1 |

LeetCode 이 문제의 제약 조건 `-2^31 <= n <= 2^31 - 1`은 **signed 32비트 정수**를 의미하는 거야. Java나 C++의 `int`가 바로 이 범위지.
