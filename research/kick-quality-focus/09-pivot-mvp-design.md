# KICK 피봇 MVP 설계: "커피 취향 프로파일링 + 매칭"

> **기반 분석**: 00-synthesis (7개 리포트 종합) + 08-deep-research (차별화 조사)
> **설계일**: 2026-02-16
> **피봇 핵심**: "메뉴 인기도 랭킹" → "개인화 맛 취향 매칭"

---

## 1. 피봇의 근본 논리

```
네이버의 강점: 거래 데이터 (POS로 "뭐가 팔리는가")
KICK의 강점:  경험 데이터 (사람만 "어떤 맛인가" 수집 가능)

"아메리카노 50잔 팔림"은 POS가 알려주지만
"이 아메리카노는 산미가 밝고 과일향이 난다"는 사람만 알려줄 수 있다.

이것이 네이버가 절대 복제할 수 없는 KICK의 유일한 방어선.
```

### 기존 vs 피봇 후

| | 기존 KICK | 피봇 후 KICK |
|---|---|---|
| **핵심 질문** | "이 메뉴가 인기인가?" | "이 메뉴가 **내 입맛에** 맞는가?" |
| **경쟁자** | 네이버 POS (직접 경쟁) | **없음** (독점 영역) |
| **데이터** | 좋다/최고/인생 (1차원) | 맛 프로필 태그 (5차원) |
| **가치** | 인기도 랭킹 | 개인화 추천 |
| **태그라인** | "커피를 평가하는 새로운 방식" | **"내 커피 취향을 발견하고, 내 입맛에 맞는 한 잔을 찾는 곳"** |

### 경쟁 포지셔닝 (피봇 후)

| | 네이버 | 카카오맵 | 구글맵 | **KICK** |
|---|---|---|---|---|
| 매장 정보 | ◎ | ◎ | ○ | △ |
| 메뉴 인기도 | ◎ (POS) | △ | △ | ○ |
| **맛 프로필** | ✗ | ✗ | ✗ | **◎ 유일** |
| **개인화 추천** | ✗ | ✗ | △ | **◎ 유일** |
| 바이럴 비주얼 | ✗ | ✗ | ✗ | **◎ 넉다운 맵** |

## 2. KICK 기존 자산 보존/변경 매핑

### 보존 (KICK 정체성 유지)

| 자산         | 보존 방식                                            |
| ---------- | ------------------------------------------------ |
| 넉다운 맵      | 그대로 유지 + **맛 필터** 레이어 추가                         |
| 3단계 리액션    | Golden Bean/Kick/Best Ever 구조 유지 + **태그 레이어** 추가 |
| "긍정 중심" 철학 | 지도에는 긍정만 표시, 부정은 대시보드에서만                         |
| 메뉴 단위 접근   | 유지, 단 "인기 메뉴" → "메뉴별 맛 프로필"로 전환                  |

### 변경

| 기존              | 변경                                    | 이유                      |
| --------------- | ------------------------------------- | ----------------------- |
| 리액션 = "좋다"      | 리액션 = "좋다 + **어떤 맛인가**"               | 정보량 10배 증가, 네이버 차별화     |
| 인기도 랭킹          | **취향 매칭 추천**                          | POS 데이터와 직접 경쟁 회피       |
| Best Ever 30+10 | **10+3**                              | 98%+ 이탈 → 70-80% 이탈로 개선 |
| 24h 소멸          | **48h 페이드아웃** (v0.1), 데이터 축적 후 24h 전환 | 콜드스타트 완화                |
| 부정 피드백 없음       | **"아쉬웠어요"** 추가 (대시보드에서만 노출)           | B2B 진단 가치 + 소비자 신뢰      |

## 5. MVP v0.1: "맛 취향 발견" (M1-M3, ~11주)

### 핵심 가설
> "커피 맛 태그가 붙은 넉다운 맵이 스페셜티 커피 매니아에게 바이럴 되는가?"

### P0 기능 3개

#### Feature 1: 맛 프로필 리액션

**사용자 플로우 (카페 현장, 총 10-15초)**:
```
QR 스캔 → 카페 자동 인식
         → 메뉴 선택: [아메리카노] [라떼] [핸드드립] [직접입력]
         → Golden Bean 탭 ☕ (커피콩 떨어지는 애니메이션)
         → 맛 태그 선택 (1-2개, 건너뛰기 가능):
           [산미 좋음 🍋] [바디 풍부 🫘] [단맛 좋음 🍯]
           [향이 좋음 🌸] [밸런스 좋음 ⚖️]
         → 완료 (넉다운 맵에 반영)
```

**맛 태그 설계 원칙**:
- 5개만 (SCA 플레이버 휠은 v0.2 이후)
- 이모지 + 한글 (커피 비전문가도 직관적)
- 복수 선택 가능, 건너뛰기 가능 (마찰 최소화)
- 태그 선택 시 "내 맛 프로필 업데이트됨!" 피드백

| 태그 | 의미 | 아이콘 |
|------|------|--------|
| 산미 좋음 | 밝은 과일 산미 | 🍋 |
| 바디 풍부 | 묵직한 풍미 | 🫘 |
| 단맛 좋음 | 자연스러운 단맛 | 🍯 |
| 향이 좋음 | 꽃/과일/초콜릿 향 | 🌸 |
| 밸런스 좋음 | 모든 맛의 조화 | ⚖️ |

**개발 기간**: 2-3주

#### Feature 2: 넉다운 맵 + 맛 필터

**메인 화면 (풀스크린 맵)**:
```
┌─────────────────────────────────────────┐
│  [전체] [산미🍋] [바디🫘] [단맛🍯] [향🌸]  │  ← 필터 바
│                                          │
│         🫘          ☕                    │
│    ☕        ☕☕☕        🍋              │
│         ☕              ☕               │  ← 넉다운 맵
│    🌸        ☕    ☕                    │     (마커 = Golden Bean)
│         ☕                ☕             │
│                   ☕         ☕          │
│                                          │
│              📍 내 위치                   │
└─────────────────────────────────────────┘
```

- 필터 선택 시: 해당 태그 비율 높은 카페 **강조**, 낮은 카페 **페이드**
- 마커 크기: 리액션 수에 비례
- 마커 탭 → 카페 상세: 메뉴별 맛 태그 분포 (바 그래프)

```
[FELT Coffee 성수점]
━━━━━━━━━━━━━━━━━━━━
라떼 (18개 리액션)
  산미 ████████░░ 60%
  바디 ██████░░░░ 40%
  향   ████░░░░░░ 25%

아메리카노 (15개 리액션)
  밸런스 ████████░░ 55%
  산미   ██████░░░░ 30%
━━━━━━━━━━━━━━━━━━━━
```

**기술**: Mapbox GL JS + supercluster (클러스터링) + 커스텀 마커
**개발 기간**: 3-4주 (성능 최적화 포함)

#### Feature 3: QR 웹 접근

- 각 카페에 고유 QR: `https://kick.coffee/r/{cafe_id}`
- **앱 설치 불필요** — PWA로 즉시 접근
- 비로그인 플로우: 첫 방문 시 닉네임만 입력
- 3회 이상 방문 시 "홈 화면 추가" 유도
- QR 스티커: 카페 테이블/카운터에 부착

**개발 기간**: 1-2주

### v0.1 개발 일정 (2인 팀)

| 주차 | 개발자 | 기획자/마케터 |
|------|--------|-------------|

| W5-6 | Feature 2: 넉다운 맵 기본 | 시드 데이터 수집 (성수동 15곳 추가) |
| W7-8 | Feature 2: 맛 필터 + 카페 상세 | 앵커 카페 15-20곳 QR 배치 |

### 카페 온보딩 (사장님 부담 = 0)

```
KICK 팀이 하는 것:
  1. 카페 방문 → 카페 정보 미리 등록 (이름/주소/메뉴)
  2. "QR 스티커를 테이블에 붙이기만 하세요" → 끝
  3. 첫 리액션 데이터가 쌓이면 대시보드 링크 전달

사장님이 하는 것:
  v0.1: QR 스티커 유지 (그 외 0 입력)
  v0.2: (선택) 대시보드 확인 (주 1회, 3분)
  v0.3: (선택) 싱글 플레이어 도구 사용
```

## 6. MVP v0.2: "내 취향 매칭" (M4-M6)

### 핵심 가설
> "맛 프로필 기반 개인화 추천이 7일 리텐션 15% 이상을 만드는가?"

### 추가 기능 4개

#### 1. 내 맛 프로필 (자동 생성)

사용자가 태그를 10개 이상 남기면 자동 생성:

```
[내 프로필]
☕ 나의 커피 취향
━━━━━━━━━━━━━━━━━
산미 ████████░░ 80%
바디 ██████░░░░ 60%
단맛 ████░░░░░░ 40%
향   ██████████ 100%
밸런스 ███░░░░░░░ 30%

🏷️ 나의 타입: "향기파 산미러버"
📊 리액션 23개 | 카페 12곳 | Best 3곳
```

**타입 분류 (규칙 기반)**:
| 타입 | 조건 | 설명 |
|------|------|------|
| 산미러버 | 산미 > 70% | 밝고 과일적인 맛 선호 |
| 바디파 | 바디 > 70% | 묵직하고 풍부한 맛 선호 |
| 향기파 | 향 > 70% | 꽃/과일/초콜릿 향 선호 |
| 밸런서 | 밸런스 > 60% | 조화로운 맛 선호 |
| 스위티 | 단맛 > 60% | 부드럽고 달콤한 맛 선호 |
| 복합형 | 2개 이상 50%+ | "향기파 산미러버" 등 조합 |

**공유 기능**: 프로필 카드 이미지 생성 → 인스타 스토리 공유 (바이럴 트리거)

#### 2. 취향 매칭 추천

```
[오늘의 추천]
"너와 비슷한 취향 47명이 좋아한 카페"
━━━━━━━━━━━━━━━━━━━━━━━━━
🏆 FELT Coffee 성수
   라떼 → 산미 72% | 향 65%
   "네 취향과 87% 일치"

🥈 Fritz Coffee 성수
   핸드드립 → 산미 85% | 향 80%
   "네 취향과 82% 일치"
```

**추천 알고리즘 (단계적)**:

| Phase | 데이터 규모 | 방법 | 정확도 |
|-------|-----------|------|--------|
| 1 (v0.2) | < 1,000 리액션 | **규칙 기반**: 코사인 유사도 (사용자 벡터 vs 메뉴 벡터) | 중 |
| 2 (v0.3) | 1,000-5,000 | **Content-Based**: 태그 벡터 + TF-IDF 가중치 | 중상 |
| 3 (Year 2) | 5,000+ | **Collaborative Filtering**: 유사 사용자 클러스터 기반 | 상 |

#### 3. Kick 도입 (하루 1회 "오늘의 최고")

- Kick 시 **맛 태그 필수** (Golden Bean은 선택적, Kick은 필수)
- 넉다운 맵에서 Kick = 더 큰 비주얼 (캐릭터 넉다운 애니메이션)
- 쿨타임: **24h 고정** (활동으로 단축 없음 — 복잡도 제거)

#### 4. 부드러운 피드백 ("아쉬웠어요")

```
리액션 선택:
  [Golden Bean 👍]  [아쉬웠어요 😐]

아쉬웠어요 선택 시:
  [산미 과함] [쓴맛 강함] [맛 없음] [기대 이하]
```

- 지도에 표시 안 됨 → **카페 대시보드에서만 노출**
- KICK의 "긍정 중심" 철학 유지하면서 B2B 진단 가치 확보
- 학술 근거: 최적 평점은 5.0이 아닌 4.2-4.5 (Spiegel Research)

## 7. MVP v0.3: "카페 인사이트" (M7-M9)

### 핵심 가설
> "카페 사장님이 '고객 맛 취향 데이터'를 경영에 활용하고, 유료 전환 의향이 있는가?"

### 추가 기능 3개

#### 1. 카페 대시보드 (Free)

```
[우리 카페 인사이트]
━━━━━━━━━━━━━━━━━━━━━━━━━

3. 핸드드립 — 8개 (향 80%, 산미 65%)

   산미 선호 45% | 바디 선호 25% | 향 선호 20% | 단맛 10%

   "라떼의 쓴맛 피드백이 늘고 있어요. 추출 레시피를 확인해보세요."
```

**네이버 POS 대시보드와의 차이**:
| | 네이버 POS | KICK 대시보드 |
|---|---|---|
| 제공 데이터 | 결제 수, 매출, 인기 메뉴 | **고객 맛 취향**, 피드백 내용 |
| 활용 | 재고/영업 관리 | **레시피 최적화, 메뉴 개발** |
| 경쟁 정보 | 없음 | **"동네 카페 고객은 산미 선호 60%"** |

#### 2. Best Ever (완화 허들: 10+3)

| | 현행 | 완화 후 |
|---|---|---|
| 조건 | GB 30 + Kick 10 | **GB 10 + Kick 3** |
| 달성 기간 | 3-6개월 | **2-4주** |
| 도달 사용자 | ~2-3% | **~20-30%** |

중간 마일스톤: 5개 → "커피러버" 배지, 15개 → "커피헌터" 배지, 25개 → "커피마스터"

#### 3. 카페 싱글 플레이어 모드

B2C 데이터 없이도 카페에게 독립적 가치를 제공:
- **오늘의 원두 기록** (무료): 사용 원두, 로스팅 날짜, 산지 메모
- **메뉴 관리** (무료): 메뉴 등록/수정, 가격 관리
- **오늘의 일지** (무료): 추출 상태 한 줄 메모

선례: OpenTable (예약 관리 SaaS 먼저 → 양면 플랫폼), Substack (작가 도구 먼저 → 독자 유입)

## 10. 솔직한 한계점

### 한계 1: v0.1만으로 리텐션 부족
- 맛 태그 + 넉다운 맵 = "재미있는 첫 경험"이지만 "매일 올 이유" 부족
- v0.2의 "내 맛 프로필 + 추천"이 나와야 진짜 리텐션 루프 시작
- **대응**: v0.1 목표를 리텐션이 아닌 "태그 수집률 + 바이럴 가능성"으로 한정

### 한계 2: 맛 태그의 주관성
- "산미 좋음"의 기준은 사람마다 다름
- **대응**: 이것이 오히려 장점. "주관적 취향"이 개인화 추천의 기반. 객관적 평가가 아닌 "내 느낌"을 기록하는 것으로 포지셔닝

### 한계 3: 커피 메뉴의 낮은 다양성
- Untappd 맥주 수만 종 vs KICK 커피 메뉴 10-20개
- **대응**: "카페 × 메뉴 × 맛"의 조합으로 다양성 창출. "FELT의 라떼 산미"와 "Fritz의 라떼 바디"는 다른 경험. 스페셜티 카페의 시즌 메뉴/싱글오리진 변경이 추가 다양성.

### 한계 4: 추천 데이터 축적 시간
- v0.2 추천 작동에 최소 1,000+ 리액션 필요
- **대응**: v0.1에서 3개월 축적. 시드 50-100개가 초기 추천 기반

### 한계 5: 하이퍼로컬 스케일
- 성수 → 연남 → 합정... 지역마다 재시딩 필요
- **대응**: 프랜차이즈 B2B 계약이 전국 확장의 트리거 (50매장 한번에 온보딩)

## 11. 설계 핵심 철학

```
1. 네이버가 할 수 있는 것을 하지 않는다   (인기 메뉴 랭킹 ✗)
2. 네이버가 할 수 없는 것을 한다           (맛 프로필 + 취향 매칭 ✓)
3. KICK의 기존 자산을 버리지 않는다        (넉다운 맵 + 3단계 리액션 유지)
4. 2인 팀이 할 수 있는 것만 한다           (Year 1은 B2C만, 3기능만)
5. 사장님에게 아무것도 요구하지 않는다     (0 입력 온보딩)
6. 검증 안 되면 빠르게 피봇한다            (3개월 단위 GATE)
```
